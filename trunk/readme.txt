-- KSNK-CMS
CMS проектов для Xilen

Технический проект, содержащий ядро CMS и админку.

Конкретные проекты игнегрируют части ядра в свой дистрибутив
для сборки используется проект PREPROCESSOR


Что касается конкретных пожеланий:

> который и модем не нагрузит и браузер любой увидит адекватно.
На стороне сервера предполагается PHP 5++ mysql 4.5++, разработка системы 
управления сайтом на других языках не планируется.

>- закрытая на штппс для регистрируемых клиентов с управлением корзиной. 
> что-то вроде ОЗОНА
? собственную корзину сейчас можно проссматривать и редактировать 
(изменять количество товара-выкидывать) 

К сожалению, закрытие части контента на https не входит непосредственно в 
систему управления. С этим нужно разбиратся с предполагаемым хостером.
С точки зрения CMS протокол общения с клиентом (http или https) выглядит прозрачно,
никаких особых дополнительных модификаций не требуется.

>- продуманная оплата картами всеми нужна, электронными деньгами - 
> традиционными  инструментами
Систему оплаты кредитными картами разумнее делать через посредников,
к примеру - ASSIST, с которым работает ozon. Напрямую с системой работать 
нерационально - очень долгий и сложный путь регистрации. С условиями подключения к 
ASSIST можно ознакомится у них на сайте. Существуют другие посредники с другими условиями.
Электронные деньги (yandex + webmoney) может быть реализовано. 

В любом случае, наличие https не требуется для работы с оплатой заказов, так как секретная 
информация (номера карточек и счетов) будет идти чеез сайты платежных систем и сайт-посредник, 
которые уже сами заботятся о секретности канала связи.  
 
>- ну, и удобная админ зона непосредственно для работы с базами товаров, 
>заказами и клиентами

Удобство можно проверить при встрече.

Существующая сейчас CMS включает в себя:
-- online-регистрация пользователей с подтверждением регистрации администратором.
-- демонстрация новостей, с импортом новостей через RSS
-- вопрос-ответ. Вопрос появляется на сайте после проверки администратором.
-- написать нам. каждый вопрос дублируется на e-mail администратора
-- блок голосования

Система администрирования включает в себя
-- создание пользователей и распределение прав пользователей
-- подтверждение online-авторизации пользователей.
-- менеджер загруженных файлов 
-- новостной блок позволяет редактировать и добавлять новости на сайт

Меню сайта с достаточной гибкостью пополняется и изменяется. Каждый пункт меню 
определяет страницу сайта. Наполнение страницы производится в системе администрирования.

Магазин включает в себя, в частности, систему импорта списка товаров с помощью подгружаемых 
csv файлов фиксированной структуры(генерируемых 1С, к примеру.).

Возможно редактирование товаров вручную поэлементно в системе управления.
