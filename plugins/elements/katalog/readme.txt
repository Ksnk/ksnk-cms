Элемент каталог

ТЗ:
-- требуется организовать схему эффективного хранения информации о товаре с возможностью 
привязки к товару:
- отдельной статьи с описанием товара
- списка фотографий

эффективность должна достигаться при выводе таблицы товаров, В каждой строчке таблицы 
будет присутствовать
- одна (первая?) фотография + ссылка на доп описние.
- характеристика товара - список подчиненных товаров, если есть.

Итого, в структуру таблицы товаров  нужно включить поля
-- фотография - ссылка на фото. которое выводится в табличном виде
-- статья - адрес статьи, которая выводится
фотографии товара располагаются в элементе фото.


При вставке элемента в админку с элементом не связывается ни один реальный элемент из списка товаров

Связывание элемента с товаром происходит только после заполнения полей 
-- артикул
-- название 
с последующим подтверждением выбора товара пользователем.

доп описание - дополнительная страничка с информацией. страничка привязана к товару, до связывания элемента 
пункт должен отсутсвовать

макет:

-- полем в таблице каталог к товару привязывается элемент - каталог. (вместо поля article)
-- в этот элемент вставляются фотографии и статья. Он и отображается в админе.

-- существует общий элемент - каталог содежащий комплект товаров, который умеет сортировать элементы.
он содержт список ID товаров или атегорию
-- два рядомстоящих элемента будут слиты в один при перемещении

-- при указании-изменении категории товара в статье автоматически добавляется-изменяется категория в последнем 
элементе
